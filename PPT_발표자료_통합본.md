```
"이질적인 4개 데이터를 어떻게 통합했는가?"

✓ 행정동 코드 매핑 (25개 구 통일)

✓ NaN 값 적절한 처리
→ 완전한 구 프로필 24개 생성
```

## **2. 2단계 평가의 필요성**
```
"왜 Level 1과 Level 2를 분리했는가?"

Level 1만: 모두에게 같은 결과 (개인화 X)
Level 2만: 품질 고려 안 함 (부정확)

Level 1 × Level 2 = 균형있는 추천
```

## **3. 실제 데이터의 힘**
```
"가상 데이터와 무엇이 달랐는가?"

마포구 20대 학생 1위 (가상)
→ 관악구 1위 (실제, 실제 20대 36.8%)

추측 vs 데이터 기반 의사결정
```

## **4. 확장 가능성**
```
"어떻게 발전시킬 수 있는가?"

현재: 24개 구 × 3개 페르소나
→ 424개 동 × 무한 사용자 프로필
→ 실시간 업데이트
→ 웹/앱 서비스화
```

# 💬 **예상 질문 & 답변**

## **Q1: 왜 25개 구인데 결과는 24개인가요?**
```
A: 양천구의 경우 Level 1 데이터는 있지만,
   실제 카드 소비 데이터 (파일 1)에서
   일부 누락이 있어 Level 2 매칭에서 제외되었습니다.
   향후 데이터 보완 예정입니다.
```

## **Q2: 코사인 유사도를 왜 사용했나요?**
```
A: 소비패턴은 "방향성"이 중요합니다.
   예: [식료품 33%, 교통 18%, ...]
   
   유클리드 거리: 절대값 차이
   코사인 유사도: 패턴 유사성 ✓
   
   → 소비 "비율" 비교에 더 적합
```

## **Q3: Level 1과 Level 2 가중치는 어떻게 정했나요?**
```
A: Level 1 4개 지표:
   - 상업활동 30%: 가장 기본적
   - 특화도 25%: 차별화 요소
   - 경제력 25%: 성장 가능성
   - 인구통계 20%: 보조 지표
   
   Level 2 4개 지표:
   - 인구통계 40%: 가장 중요 (같은 연령대)
   - 소비패턴 35%: 라이프스타일
   - 소득수준 15%: 가격대 적정성
   - 업종선호 10%: 세부 조정
   
   → 도메인 지식 + 실험 결과 기반
```

## **Q4: 강남구가 7위인 이유는?**
```
A: 강남구의 특징:
   - 상업활동: 70.9 (높음)
   - 특화도: 51.4 (낮음!) ← 핵심
   
   특화도가 낮다 = 여러 업종 골고루
   → 뚜렷한 강점 없음
   → 종합점수 하락
   
   BUT 특정 사용자에게는 적합할 수 있음
   → 개인화의 중요성
```

## **Q5: 실제 서비스로 만들 계획은?**
```
A: 3단계 로드맵:
   
   Phase 1 (현재)
   - 24개 구 × 3개 페르소나
   - 오프라인 분석
   
   Phase 2 (6개월)
   - 424개 동 확장
   - 웹 대시보드 개발
   - 실시간 API 연동
   
   Phase 3 (1년)
   - 모바일 앱
   - AI 추천 고도화
   - B2C/B2B 서비스
```
